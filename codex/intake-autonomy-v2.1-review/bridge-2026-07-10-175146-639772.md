**MF-3 — Reviewer output contract: objection schema is undefined**

§3 RD3, §4 stage 5, §4 components: *"reviewer output contract (structured objections)"* listed as a build item with no schema.

`intake-retry-gate.js` validates `objections[].resolvedBy` — but what shape must the reviewer *emit* for `objections[]` to exist? ### SHOULD-FIX — 4 findings

---

**SF-1 — `world-absent` assignment authority: undefined; creates a taint-bypass path for first-run candidates**

§3 RD2: "`world-absent` — assignable only with a recorded complete source sweep." Who assigns it?

A first-run untainted candidate (no NO-GO on record) whose researcher self-reports `world-absent` with a low-effort source sweep bypasses `research-defect`'s one-pass bound and instead gets a weekly clock ×4. "Evidence genuinely not published" is a judgment call, not a deterministic check. The taint invariant doesn't block this because the candidate has no prior NO-GO.

*Patch direction:* Specify that `world-absent` is assigned by `validate-candidate-evidence.js` only — after a structured `absence-rationale` passes RD1's three-component check AND the sweep satisfies a defined completeness predicate. The researcher cannot self-select it.

---

**SF-2 — `research-defect` one-repair bound: no enforcement field in sidecar schema**

§3 RD2: "×1 → `decision-required` + notify." §7: "RD2 lane test: exactly ONE repair pass."

The sidecar schema (RD5) has `attempts[]` and `class` but no `repairAttempts` counter. If state-dir is cleaned between runs, or the parked-store reads a v1 sidecar, the ×1 bound has nothing to check.

*Patch direction:* Add `repairAttempts: number` to the RD5 sidecar schema. `intake-parked-store.js` increments it on retry-sweep entry for `research-defect` candidates and reclassifies to `decision-required + notify` when `repairAttempts >= 1` before allowing re-entry.

---

**SF-3 — §8 K2 SE acceptance test implies automatic execution; it requires an undeclared owner action**

§8 criterion 1: "K2 SE is the end-to-end acceptance test. Under v2.1 it either ships… or parks for a **different, better** reason."

Per RD5, K2 SE migrates to `decision-required` — never auto-retried. The pipeline will not automatically process K2 SE. The test is untestable as written without owner authorization.

*Patch direction:* Add precondition to criterion 1: "After the owner issues an explicit re-attempt instruction (the sanctioned RD2 event trigger for `decision-required`)…"

---

**SF-4 — `needs-source-resolution:conflicting` in the taxonomy table conflicts with the producer-compatibility note**

§3 RD2 table lists `:conflicting` as a current `judgment-on-evidence` reason. The same section's producer-compatibility note says "The `:missing` / `:conflicting` subtypes do **not exist** until the producer contracts are amended." Building `intake-park-taxonomy.json` from this table includes a reason that can never be triggered.

*Patch direction:* Mark `:conflicting` in the table as "(future — pending producer contract amendment)" so the taxonomy config correctly omits it until the runbook is updated.
