**MEDIUM — case-insensitive FS bypass on macOS** (`assertSafeOutDir`, line 548)

`abs.startsWith(ROOT + path.sep)` is a byte-exact string comparison. On APFS/HFS+ (the dev machine), `--out /Users/mustafaozturk-macmini/dev/Claude/Projects/3DPrintAssistant/data/` resolves literally with the capital letters; `insideRepo` is `false`; the guard returns safe. The write lands in `data/` which is served as Cloudflare assets. This is a real bypass on macOS for any path whose case differs from ROOT.
**Fix direction:** `fs.realpathSync` on both `abs` and `ROOT` before comparing, or `path.resolve` + `toLocaleLowerCase()` for both sides on macOS. Note: `realpathSync` throws if the dir doesn't exist yet (pre-mkdir), so resolve the parent chain.

---

**LOW — symlink at an outside-repo path bypasses guard** (`assertSafeOutDir`, line 549)

`path.resolve` is a pure string op; it does not dereference symlinks. `--out /tmp/link-to-repo-data` is accepted (out-of-repo), but `fs.mkdirSync` + `fs.writeFileSync` follow the symlink into the repo. Low probability for a single-owner CLI but worth noting given the PII risk.

---

**LOW — TC21 missing error-message assertion** (`printer-intake-scout.test.js`, line 267)

TC19 asserts both exit 2 *and* `/staging|refusing --out/i` in stderr. TC21 checks exit 2 only — a crash or a wrong-reason exit would pass. The asymmetry weakens TC21 as a refusal proof.
**Fix direction:** add `check('error explains refusal', /staging|refusing --out/i.test(r.stdout + r.stderr), ...)` in TC21, mirroring TC19.

---

**LOW — ACCEPT branch (outside-repo paths) has zero test coverage**

TC19, TC20, TC21 cover refuse/approve inside-repo. There is no TC that passes `--out /tmp/some-path`, exits 0, and verifies artifacts are written. If the guard ever gains a regression that refuses all paths, the test suite would not catch it.
