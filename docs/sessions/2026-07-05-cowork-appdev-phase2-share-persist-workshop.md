## Durable context

- **One codec is the cross-platform contract.** Web `state-codec.js` FIELDS table is the single source of truth for state serialization (localStorage `3dpa_state_v1`, share URLs, Workshop profiles). iOS mirrors it in `Models/AppStateWebCodec.swift` (web-verbatim keys incl. snake_case advanced fields, same field order, same unknown-id degradation). Adding a state field = one row in each; the round-trip tests pin both.
- **The Workshop backup envelope is byte-compatible across platforms by test.** iOS `WorkshopStore` renders `{"v":1,"profiles":[...]}` exactly like `JSON.stringify(env, null, 2)` via an ordered emitter; `WorkshopStoreTests` imports a fixture generated by the real web `workshop-store.js` and asserts byte-identical re-export. If either side changes the format, regenerate the fixture from web (node snippet in the ledger/test header) — web stays master.
- **iOS journal (IMPL-044 W2) deferred deliberately**: iOS has no troubleshooter surface, so the symptom deep-link half of the feature is unbuildable there today. Journal data still survives iOS import/export untouched (emitter handles `journal` entries). Revisit when a troubleshooter view ships on iOS.
- **iOS has no localization infrastructure** (`Utils/Strings.swift` hardcoded English by design, "swap for NSLocalizedString later"). The "EN+DA for new copy" requirement was deviated on to avoid a mixed-language UI; recorded in the ledger.
