# iOS 1.1.0 Notification Release Implementation Plan

**Status:** Owner-ratified draft `1a0015f`; authenticated hostile review returned
`GO-WITH-PATCHES`, and every accepted P1/P2 is committed separately.
Convergence review is pending; implementation remains unauthorized.

> **For agentic workers:** REQUIRED SUB-SKILL: Use
> `superpowers:executing-plans` to implement this plan task-by-task in the
> controller session. Invoke `superpowers:using-git-worktrees` only if the
> execution workspace is dirty or parallel work makes isolation necessary.
> Steps use checkbox (`- [ ]`) syntax for tracking. Do not use subagents unless
> the owner explicitly changes the no-delegation boundary.

**Goal:** Ship iOS 1.1.0 with explicit opt-in APNs notifications for newly
supported printers and meaningful app updates, issues #2–#5, visible native
Bambu/Orca/Prusa exports, the nine existing local printer/export commits, and
complete privacy, operations, TestFlight, and App Store evidence.

**Architecture:** The existing Cloudflare Worker remains the single first-party
boundary. It gains source/config denial, EU D1 consent and delivery ledgers,
Queue/DLQ fan-out, APNs token encryption, and a local-only owner CLI; public
send stays feature-flagged off. iOS adds a `UserNotifications` delegate,
observable permission/registration service, typed payload coordinator, and
engine-ready route consumption while preserving an immutable shared catalog
snapshot. Existing issue fixes and native export wiring stay separate commits
inside the same gated 1.1.0 train.

**Tech Stack:** Swift 5.9, SwiftUI, UIKit, UserNotifications, CryptoKit,
XCTest/XCUITest, JavaScript ES modules, Cloudflare Workers, D1, Queues, Web
Crypto, Wrangler 4.36+, Vitest 4.1+, XcodeGen, Fastlane, GitHub Actions, App
Store Connect.

## Global Constraints

- The owner ratified the design on **2026-07-18**. This plan does not itself
  authorize implementation, external provisioning, an iOS push, TestFlight, or
  App Store submission; each named owner gate still applies.
- Public App Store is **1.0.4**. The complete release is **1.1.0**; the unstarted
  1.0.7/1.0.8 sequence is superseded. Tip jar remains parked and is recommended
  for 1.2.0 at its future gate.
- Both notification topics default off: `new_printers` and `app_updates`.
- The system prompt appears only after the user selects at least one topic and
  taps **Enable selected notifications**. Request only `.alert` and `.sound`.
- Push is alert-only. Do not add Background Modes, badge, critical, or
  time-sensitive notification capability.
- Maximum one non-cancelled public campaign in any rolling seven-day window;
  canaries do not count. `PUSH_PUBLIC_SEND_ENABLED=false` until the owner sends
  deliberately after public release.
- `PUSH_REGISTRATION_ENABLED=false` must fail closed for register while
  unregister remains available and idempotent.
- APNs 2xx means **accepted by APNs**, never delivered or opened.
- Never log or persist raw APNs tokens outside encrypted D1 storage and the
  protected on-device deletion/rotation handle. Never log secrets,
  Notification IDs, ciphertext, authorization headers, notification opens, or
  generated profile data.
- `IOS_PUSH_REGISTRATION_SECRET` is distinct from
  `FEEDBACK_HMAC_SECRET`; it is abuse friction, not device identity.
- Dedicated APNs `.p8`, admin bearer, registration HMAC, and encryption keys
  enter Cloudflare/GitHub only through secure secret entry. They never enter
  git, docs, screenshots, review prompts, shell history, or chat.
- Raw token primary key is `(environment, token_hash)`; all delivery and canary
  lookups include `development|production`. A 16-hex Notification ID must match
  exactly one row or be rejected as ambiguous.
- A provider-auth APNs 403 blocks the campaign, preserves every device row,
  persists the unchanged cursor, sends it to DLQ, and stops later sends in that
  batch.
- Rotation uses one atomic D1 batch: upsert new route, change only
  `pending|retryable` deliveries to distinct `token_rotated`, then delete the
  prior route. Database transition triggers increment counts exactly once.
- Unregister deletes the device row and device-specific delivery rows
  immediately. Daily retention deletes delivery rows after 30 days and stale
  devices after 180 days.
- Web is master for `engine.js` and `data/`. This release authorizes no new web
  engine/data behavior. Every final gate still proves byte identity and runs
  both platform harnesses because nine existing iOS mirror commits ship.
- Output export is engine-selected with no manual override:
  `bambu_bundle`, `orca_bundle`, or `prusa_bundle`. The share sheet opens only
  after every expected file is written.
- One owner finding or accepted review finding per commit. iOS commits remain
  local until the entire 1.1.0 train, tests, archive entitlement, canary, copy,
  and hostile review are green and the owner approves the single push.
- `CURRENT_PROJECT_VERSION` remains Fastlane-owned. Change
  `MARKETING_VERSION` to `1.1.0` during local implementation before final gates.
- No task is complete with uncommitted changes in a touched repo. Preserve the
  pre-existing nine-commit iOS train; never replay or squash it.
