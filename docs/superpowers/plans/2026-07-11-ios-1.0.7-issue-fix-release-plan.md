# iOS 1.0.7 Issue-Fix Release Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use
> `superpowers:executing-plans` to implement this plan task-by-task in the
> controller session. Do not dispatch subagents unless the owner explicitly
> changes the current no-delegation boundary. Steps use checkbox syntax for
> tracking.

**Goal:** Fix GitHub issues #2–#5 on iOS, land the narrow web analytics support
they require, and produce a new owner-testable build on the existing 1.0.7
TestFlight train.

**Architecture:** `WorkshopStore` becomes the source of truth for normalized
saved-state and filtered backup export. `OutputView` and `WorkshopView` expose
visible, slicer-honest actions and emit typed aggregate analytics. Claude's
web-only branch `origin/claude/cold-start-3dpa-issues-ihs5du` is treated as
validated reference behavior, not proof that iOS is fixed: reconcile each web
delta against current `origin/main`, reuse only still-needed current-main-safe
work, and implement the remaining iOS mirror directly. Shared engine/data files
are not edited; the existing K2 SE local iOS commit is verified as a bundled
mirror of already-live overlay data.

**Tech Stack:** Swift 5.9, SwiftUI, XCTest/XCUITest, JavaScript/Cloudflare
Workers, Node test runner, XcodeGen, GitHub Actions, Fastlane, App Store Connect.

## Global Constraints

- Public App Store is 1.0.4; keep `MARKETING_VERSION` at **1.0.7**.
- Do not edit `CURRENT_PROJECT_VERSION`; Fastlane assigns `YYYYMMDDHHmm`.
- K2 SE is already live via overlay `2026071005`; `bfc9a2b` only bundles it.
- Web is master for `engine.js` and `data/`; never make an iOS-only shared-file
  correction.
- No engine/data behavior change is planned. Because `bfc9a2b` rides the train,
  still run web validators, walkthrough, iOS tests, and byte-identity checks.
- One owner finding or accepted review finding per commit.
- iOS commits remain local until the full train is TestFlight-ready.
- Do not merge the July 9 Claude branch wholesale. Read it first, reconcile it
  against current `origin/main`, and skip/cherry-pick/adapt only the deltas that
  are still missing and current-main-compatible.
- A task is not complete while any touched repo has uncommitted or unstaged
  changes. Intentional local commits/ahead state must be explicitly documented;
  after the TestFlight/public-release flow, every touched repo must be clean.
- Current analytics detail property is `type`, not `exportType` or
  `exportScope`; the 20-blob order stays unchanged.
- No identity, free text, profile names, file contents, or generated settings in
  analytics.
- Native Orca/Prusa file serializers are out of scope; label text sharing
  honestly and keep issue #4/follow-up open for those serializers.
- Current-state correction: neither public 1.0.4 nor the earlier 1.0.7
  TestFlight build exposes export. The existing private `exportMenu` is
  unreferenced; Orca/Prusa text sharing becomes available only through Task 6.
- Do not submit to App Review before owner acceptance of the exact TestFlight
  build.
