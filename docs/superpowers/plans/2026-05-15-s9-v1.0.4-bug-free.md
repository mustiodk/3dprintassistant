## Scope decision (locked)

**In scope (real bugs):**

1. **M1 — Web engine fan_multiplier upper-bound asymmetry.** `engine.js:1246` + `engine.js:2655` guard `fan_multiplier > 0` only. After S8.7 commit `48304d3`, `scripts/validate-data.js` rejects `> 1`. Engine should match — currently a `fan_multiplier: 1.5` would silently pass and scale fan emission by 1.5× if data ever bypassed validate-data. Codex post-S8.7 audit called it "real hygiene"; reviewer surfaced as M1.
