**Architecture:** All changes happen in `engine.js` first (the canonical source of business logic), backed by data in `data/*.json` and validated by `scripts/walkthrough-harness.js` + `scripts/profile-matrix-audit.js`. ## Standing rules for every task

Every task in this plan obeys these without exception:

1. **TDD per finding.** Walkthrough assertion (+ iOS XCTest written at Phase 2) lands red before the engine change. The test is the contract — write it first, watch it fail, then implement.
2. **One finding = one commit per platform.** No bundling unrelated findings in a single commit.
3. **Provenance on every new `resolveProfile` emission.** Use the existing `S(value, why, { source, ref })` helper. Sources: `vendor` / `default` / `rule` / `calculated`.
4. **IMPL-040 chip parity.** Any new or changed chip text → prefix generator in `engine.getFilters(state)` + iOS `EngineServiceTests.swift` assertion that the chip number equals `resolveProfile` output. See `docs/specs/IMPL-040-chip-desc-parity.md`.
5. **Verification gate per finding** (in order, all must be green before commit): `node scripts/validate-data.js` → `node scripts/walkthrough-harness.js` (no `❌` rows) → `node scripts/profile-matrix-audit.js` (0 curated failures, 0 broad failures) → UI smoke for visible-to-user changes only.
6. **Per-finding checkpoint.** After commit + green verification, decide: continue or wrap. Continue if next finding looks bounded and context isn't heavy; wrap (Trigger A) at clean commit boundary otherwise.
7. **Web push to `main` is free** — Cloudflare Pages auto-deploys; that is the documented pattern.
8. **iOS push to `main` is gated** — local commits only until Phase 2 ship-ready check passes.
