## 3. JS runtime research (the load-bearing decision — Android's D5/D5a analog)

Researched mid-2026 with maintenance/size facts verified against GitHub/Maven Central; latency/memory are order-of-magnitude estimates. Constraint set: bundled trusted JS (~250 KB total, ES2020+), ~35 **synchronous** API functions (the iOS bridge is sync-on-serial-queue), multi-file loading + fetch/localStorage/console polyfills, exception capture with JS stacks, small binary, solo-dev longevity.

| Option | Verdict | Key facts |
|---|---|---|
| **QuickJS via `HarlonWang/quickjs-wrapper`** | **TOP PICK** | True sync `evaluate()` from Kotlin (direct mirror of the iOS serial-queue bridge → single-thread dispatcher); ~2.3 MB total / ~0.6 MB per ABI; ES-module + custom module-loader support; injectable `console` + host functions for fetch/localStorage polyfills; stated 16 KB-page support; QuickJS ≈ ES2023. Risk: **bus factor 1, last release Jul 2025** — mitigated by vendoring (small wrapper, healthy `quickjs-ng` upstream, frozen version safe for pure-logic eval). Interpreter speed is irrelevant at this workload. |
| `androidx.javascriptengine` 1.1.0 | Runner-up / recorded fallback | Google-maintained (stable May 2026), zero binary cost (system WebView V8). Fights the architecture twice: async-only `ListenableFuture` (sync emulation = blocking a dedicated thread + per-call IPC) and **no synchronous host-function callbacks** → localStorage/fetch polyfills genuinely awkward. Adopt only if the QuickJS path dies. |
| Javet (V8) | Ruled out | Best-maintained embedder (monthly releases, full sync, superb exceptions) but ~24 MB per ABI + minSdk 29 floor — disqualified by binary size for this workload. |
| J2V8 | Ruled out | Revived Nov 2025 after years dormant; ~15 MB/ABI; bus factor 1; V8 currency unverified. |
| Zipline | Ruled out | Solves a different problem: Kotlin/JS-compiled modules behind suspend service interfaces, not hand-written JS with a sync API. |
| Invisible WebView bridge | Ruled out | Async-only `evaluateJavascript`, swallows errors, slowest startup, heaviest memory. (This is what the parked prototype uses — as a *packaging* proof, not a bridge.) |

**Google Play policy:** no issue — the Device & Network Abuse policy restricts interpreted code *loaded at run time ("not packaged with the app")*; JS bundled in the AAB is explicitly outside the restriction (verified against current policy text).

**Decision protocol:** the spec (AD5) adopts the top pick **subject to a mandatory AG1-entry spike** (load engine.js + 3 modules, run a golden-mirror subset, verify exception traces / 16 KB alignment / minSdk floor / init latency / **which QuickJS the wrapper actually embeds (Bellard vs `quickjs-ng`) — "healthy upstream" is half the bus-factor mitigation and is unverified**, ledger-recorded GO) — same discipline as iOS D5a, because the wrapper's claims are verified-by-research, not yet verified-by-execution.

## 4. Google Play facts (owner-dependency drivers; verified mid-2026 unless noted)

1. **Personal account:** $25 one-time + government-ID verification (hours–2 days). D-U-N-S is org-only.
2. **Closed-testing requirement (personal accounts created after 2023-11-13):** **12 opted-in testers continuously for 14 days** before *applying* for production access (reduced from 20 in Dec 2024; unchanged since). Testers must genuinely engage — "insufficient engagement" is the top 2026 rejection reason. **This is the schedule long-pole → the plan opens closed testing as early as a usable build exists and recruits via the existing Discord.**
3. **EU DSA:** trader-status **declaration is mandatory** (same mechanic as the Apple block the owner hit), but a zero-monetization hobbyist declares **"not a trader"** and proceeds — no address/phone publication. (Declaration requirement verified; "non-trader is correct for zero monetization" is the likely-correct reading, owner confirms in Console.)
4. **Target API:** the official Play requirement page currently says **API 35** for new apps/updates; the researched **"API 36 from 2026-08-31" is the expected annual bump but is NOT yet verifiable on the official page** (Codex CX-3 — downgraded from verified to likely). Decision unaffected: target 36 proactively (toolchain is already SDK 36); **re-verify the requirement at AG7 entry**. **16 KB pages:** applies (the QuickJS wrapper is native code) — wrapper states support; spike verifies alignment.
5. **Play App Signing:** mandatory (AAB). Owner generates an upload keystore; Google holds the signing key; leaked upload keys are replaceable (lower stakes than Apple certs).
6. **CI:** GitHub Actions → Play is standard. Options all maintained mid-2026: Gradle Play Publisher v4 (Gradle-native), fastlane supply, `r0adkll/upload-google-play` v1.1.5 (**simplest; mirrors the manual-dispatch `testflight.yml` pattern**). Needs a GCP service account granted release permission in Play Console + JSON key secret + base64 keystore secrets. **Quirk: the first-ever upload must be manual via the Console.**
7. **Tracks:** internal (≤100 testers, near-instant, no full review, exempt from Data-safety) → closed (counts toward the 12/14 rule, reviewed-fast) → production (first-app review **~7–14 days** for a new personal account). Realistic calendar: **3–5+ weeks** from closed-test start to production live.
8. **Data safety:** server-stored anonymous events declare as "App interactions/Diagnostics, not linked, not shared"; feedback text declares as optional user-generated content. Exact mapping is a judgment call on the form (owner + assistant at the release gate).
9. **Inactivity:** Google closes dormant personal accounts (no-app-after-1-year, or low-install + no-login-180d) — fine once the app ships; don't create the account then stall for a year.

Hostile-reviewed 2026-06-05 (nav lockdown, local-404, smoke hardening — all patched). Inspected via a fresh read-only clone in this session's scratchpad; **no push made; Phase-1 no-push boundary untouched.**

**What it proved (still valuable):** the toolchain path — Android command-line SDK (build-tools 36.0.0, platform android-36, emulator + API 36 image) + Java 17, **no Android Studio required**; APK builds, installs, launches; a signed debug-keystore loop for `adb install -r`; a repeatable smoke pattern.

## 7. Android-specific risk delta (beyond the reused iOS-audit §6)

- **Greenfield has no test floor** — iOS v2 inherited 135 XCTests; Android starts at zero. Mitigation: the contract fixtures (golden mirrors, envelope, codec, op-log) are the *imported* floor — they exist before UI work and gate every session (spec §3).
- **Runtime wrapper bus factor 1** — vendored + fallback recorded (§3); the JS files themselves are engine-agnostic, so a runtime swap is bounded pain.
- **Play first-publication calendar** (12×14 closed test + first review) — owner-side long-pole; plan front-loads account creation (Gate 0) and opens closed testing mid-program.
- **Single toolchain machine** (mac-mini) — execution sessions are machine-pinned; ledger + pushed branches keep state cross-machine visible (same D1a logic as iOS).
- **Instrumented-test dependency** — mirror verification needs an emulator (local + CI); if CI emulator time proves flaky, the fallback is local-emulator gating + CI build-only (ledger-recorded).
- **Compose/Material vs brand identity** — the design system must override Material defaults to the 3dpa brand (dark #0a0a0b, green #00e5a0, Syne + DM Mono — both OFL-licensed, bundleable); risk is a generic-Material look, mitigated by the AD7 token system + screenshot packets.
